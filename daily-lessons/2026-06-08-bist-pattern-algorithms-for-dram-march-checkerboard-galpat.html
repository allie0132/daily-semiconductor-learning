BIST Pattern Algorithms for DRAM (March, Checkerboard, GALPAT)

Monday, Jun 08 2026 · Lesson 34 of 56

Why BIST Patterns Matter in DRAM

Built‑In Self‑Test (BIST) provides on‑chip pattern generation and response analysis without external ATE stimulus. For high‑density DRAM and HBM stacks, BIST reduces test time, improves yield, and isolates hard‑to‑detect faults such as stuck‑at, transition, coupling, and address decoder errors.

March Tests – Fundamentals and Common Variants

March algorithms are ordered sequences of R/W operations executed on every memory cell. The most widely used for DRAM is March C‑ and March CC‑ (JEDEC JESD79‑4B). Example of March C‑:

  • ↑(w0); ↑(r0,w1); ↑(r1,w0); ↓(r0,w1); ↓(r1,w0);

↑ denotes address increment, ↓ address decrement. Each operation is performed on all addresses in the specified order. Timing constraints (tRCD, tRP, tRAS) must be respected between successive commands; BIST controllers typically insert tCCDL (command delay) cycles automatically.

Checkerboard Patterns – Data‑Dependent Fault Detection

Checkerboard patterns write alternating 0/1 bits across word lines and bit lines, creating a high‑frequency spatial pattern. Two primary forms are:

  • Row‑Checkerboard: each row alternates 0/1, useful for detecting row‑line coupling and open‑shooter defects.
  • Column‑Checkerboard: each column alternates, targeting bit‑line coupling and crosstalk.

Implementation steps in BIST:
1. Write 0xAA…AA (1010…) to all addresses.
2. Read back and compare.
3. Write 0x55…55 (0101…) to all addresses.
4. Read back and compare.Timing must obey JEDEC tWR (write recovery) between write and the subsequent read.

GALPAT – Gaussian‑Like Pattern for DRAM

GALPAT (Gaussian‑like Adaptive Lattice Pattern) is a JEDEC‑standardized pseudo‑random pattern (JESD79‑4B §6.6) that blends deterministic and random characteristics. It is generated by a linear‑feedback shift register (LFSR) seeded with a fixed pattern, then masked with a Gaussian weight matrix to create slowly varying spatial gradients.

  • Sequence: LFSR → XOR with Gaussian mask → Write → Read → Compare
  • Fault coverage: excels at detecting address aliasing, pattern‑sensitive coupling, and marginal cell degradation because the pattern stress varies gradually across the array.

BIST engines typically provide a GALPAT_EN register; when set, the internal LFSR length (e.g., 32‑bit) and mask coefficients are selectable via GALPAT_CFG registers.

Integrating Patterns into HBM BIST Flow

HBM stacks embed a dedicated BIST block per die. The test flow is:

  1. Initialize BIST controller (set MODE=March/Checkerboard/GALPAT).
  2. Run March C‑ for address decoder validation.
  3. Execute Checkerboard to stress line‑to‑line coupling.
  4. Finish with GALPAT for high‑frequency random‑deterministic coverage.
  5. Collect error logs via ERR_LOG registers; map to physical locations using ADDR_MAP tables.

Each phase must respect HBM timing (tRCD, tRP, tCK) and the BIST clock must be synchronized to the DRAM’s internal PLL (typically 1 GHz for DDR5‑based HBM). Failure analysis uses the error address, expected pattern, and actual read value to pinpoint stuck‑at, transition, or coupling defects.

⚡ Key Takeaways

  • March tests provide systematic address‑order coverage; use March C‑/CC‑ for DRAM decoder verification.
  • Checkerboard patterns stress line‑to‑line coupling and can be implemented with simple alternating writes.
  • GALPAT combines deterministic and pseudo‑random elements, offering superior detection of marginal and address‑alias faults.
  • In HBM, sequencing March → Checkerboard → GALPAT maximizes fault coverage while staying within the tight timing envelope of the stack.
